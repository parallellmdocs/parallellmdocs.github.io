    Megatron-LM ModelOpt 蒸馏集成

    目录

    [[TOC]]

    使用方法

    前置条件

    在给定数据集上执行软标签知识蒸馏时，需要一个已充分训练的大模型作为教师，利用其 logits 为小模型提供标签。

    所需数据： * 教师模型权重 * 学生模型权重（若非从头训练） * 教师模型的 NeMo 格式配置文件 * 蒸馏运行配置文件 * 分词器 * 数据集

    此外需要安装 NVIDIA Model Optimizer 库（最低版本 0.15）。

    教师检查点格式

    教师模型参数通过 NeMo 样式的 model_config.yaml 描述，存放于教师权重根目录。该文件的内容会覆盖学生模型的命令行参数后再传入教师构造函数。Megatron-LM 默认不包含该文件，需要手动创建。

    注意：NeMo 式 YAML 中的键并不与 Megatron-LM 参数完全一一对应，转换逻辑位于 megatron/inference/gpt/model_provider.py。

    蒸馏配置文件

    蒸馏流程通过单独的 YAML 文件配置，例如：

    logit_layers: ["output_layer", "output_layer"]
intermediate_layer_pairs:
  - ["decoder.layers.0.input_layernorm", "decoder.layers.0.input_layernorm"]
  - ["decoder.final_layernorm", "decoder.layers.30.input_layernorm"]
skip_lm_loss: true
kd_loss_scale: 10.0
    • logit_layers：指定学生与教师 logits 所在的子模块名称。
    • intermediate_layer_pairs：可配置若干对中间激活计算额外损失。
    • skip_lm_loss：是否跳过原始 LM 损失，仅使用 KD 损失。
    • kd_loss_scale：若保留 LM 损失，在合并前对 KD 损失进行缩放。

    训练

    在 pretrain_gpt.py 中添加如下参数即可触发蒸馏：

    --kd-teacher-load <path-to-teacher-checkpoint>
--kd-distill-cfg <path-to-distill-config-yaml-file>
--export-te-mcore-model

    注意：若教师检查点格式与学生不同（学生格式由 --ckpt-format 指定），可额外使用 --export-kd-teacher-ckpt-format 指定教师格式。

    蒸馏 API 与设计

    蒸馏通过 NVIDIA Model Optimizer 库 实现。

    模型构建时，将学生模型包装为 modelopt.torch.distill.DistillationModel，内部包含教师模型。损失函数根据蒸馏配置文件中给定的学生与教师模块名生成。

    默认情况下，logits 使用 KL 散度损失，中间特征使用余弦相似度，定义在 megatron/inference/algos/distillation.py。

    限制

    • 暂不支持交错流水线并行。
    • 仅支持 Megatron-Core 模型（不支持 Legacy 版本）。

    已知问题

    • 转换为 modelopt.torch.distill.DistillationModel 时，每个微批会额外占用少量显存。如启用 --manual-gc，可能在若干步后触发 OOM。
    • 存在 CUDA 内核问题：与单独运行学生模型相比，学生前向延迟明显增加，训练迭代时间可能比预期长约 40%。
