    新版分词器系统

    与旧版分词器系统的主要区别

    1. 类似 Hugging Face 的 API

    我们新增了 MegatronTokenizer 类，提供与 Hugging Face 类似且易于使用的 API：

    .from_pretrained() —— 从目录或文件加载分词器，可自动检测类型与配置。

    .write_metadata() —— 保存分词器配置（元数据），便于后续直接复用，而无需再次指定参数。

    这样就无需在训练脚本中填写长串初始化参数或硬编码设置。

    2. 分词器元数据

    现在使用 JSON 元数据文件集中存放分词器的关键配置： - 分词器库（如 HuggingFace、SentencePiece、TikToken 等） - 聊天模板 - 分词器类

    优势： - 相关参数仅需设置一次。 - 不再需要在命令行中传递多个分词器配置参数。 - 便于共享——复制分词器目录及其元数据文件即可。

    3. 库类现已内部管理

    旧版中，用户必须了解应使用的分词器库（如 SentencePieceTokenizer、HuggingFaceTokenizer 等）并手动实例化。

    新版系统中： - 库类型会根据元数据自动识别。 - 底层会选择正确的分词器实现。 - 使用者无需手动处理分词器类。

    3. 支持模型特定的分词器类

    系统现在支持： - 内置的 LLM 专用分词器。 - 自定义分词器：通过继承 MegatronTokenizerText 创建自定义类，并在元数据文件的 tokenizer_class 字段中声明即可。 - 在保持默认设置简洁的同时，为高级定制提供空间。

    4. 使用方式

    创建并保存元数据

    from megatron.core.tokenizers import MegatronTokenizer

# Metadata is written as tokenizer_metadata.json inside the tokenizer directory.
MegatronTokenizer.write_metadata(
    tokenizer_path="/path/to/tokenizer.model",
    tokenizer_library="sentencepiece",
    chat_template="chat template in jinja format",
)

# Example: register a custom tokenizer class
from megatron.core.tokenizers.text import MegatronTokenizerText

class CustomTokenizer(MegatronTokenizerText):
    ...

MegatronTokenizer.write_metadata(
    tokenizer_path="/path/to/tokenizer.model",
    tokenizer_library="sentencepiece",
    chat_template="chat template in jinja format",
    tokenizer_class=CustomTokenizer,
)

# Save metadata to another location
MegatronTokenizer.write_metadata(
    tokenizer_path="/path/to/tokenizer.model",
    tokenizer_library="sentencepiece",
    metadata_path="/path/to/save/metadata.json",
)

    恢复分词器

    from megatron.core.tokenizers import MegatronTokenizer

MegatronTokenizer.from_pretrained(
    tokenizer_path="/path/to/tokenizer.model",
)

# If metadata is stored elsewhere
MegatronTokenizer.from_pretrained(
    tokenizer_path="/path/to/tokenizer.model",
    metadata_path="/path/to/metadata.json",
)

# Pass metadata as a dictionary
MegatronTokenizer.from_pretrained(
    tokenizer_path="GPT2BPETokenizer",
    metadata_path={"library": "megatron"},
    vocab_file="/path/to/vocab.txt",
)

# Provide additional parameters
MegatronTokenizer.from_pretrained(
    tokenizer_path="/path/to/tokenizer/model.json",
    metadata_path={"library": "tiktoken"},
    pattern="v2",
    num_special_tokens=1000,
)

# Null tokenizer
MegatronTokenizer.from_pretrained(
    metadata_path={"library": "null"},
    vocab_size=131072,
)

    4. Megatron-LM 预训练兼容性

    新版分词器系统与 Megatron-LM 预训练脚本兼容。若未指定 --tokenizer-metadata，会自动生成默认元数据文件。

    # Null tokenizer
torchrun --nproc_per_node=1 pretrain_gpt.py \
    ... \
    --tokenizer-type NullTokenizer \
    --vocab-size 131072

# HuggingFace tokenizer with explicit metadata
torchrun --nproc_per_node=1 pretrain_gpt.py \
    ... \
    --tokenizer-type HuggingFaceTokenizer \
    --tokenizer-model meta-llama/Meta-Llama-3-8B \
    --tokenizer-metadata /path/to/metadata.json

    Megatron-LM 预训练脚本仍支持旧版分词器系统。若需启用旧版，只需添加 --legacy-tokenizer 参数。
