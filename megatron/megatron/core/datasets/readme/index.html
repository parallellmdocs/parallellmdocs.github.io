    数据管线

    数据预处理

    数据预处理围绕以下类构建：

    1. IndexedDatasetBuilder
    2. IndexedDataset

    目前端到端的数据预处理流程仍需用户自行实现，更多细节可参阅类的文档字符串。

    IndexedDatasetBuilder

    IndexedDatasetBuilder 负责构建并合并多个 IndexedDataset 实例。

    IndexedDataset

    IndexedDataset 是 Megatron Core 中最低层的数据接口。内部引用两个二进制文件：数据文件（.bin）存储文档/序列内容，索引文件（.idx）存储文档/序列的元数据。

    索引文件首先保存数据集级别的元信息： - 索引头（用于兼容旧版本） - 索引版本（用于兼容旧版本） - 写入数据文件时使用的数据类型编码 - 数据集中的序列数量 - 数据集中的文档数量

    随后记录文档级与序列级元数据： - 按顺序记录每个序列包含的元素数量 - 按顺序记录每个序列的字节偏移量 - 按顺序记录每个文档对应的连续序列区间 [...) - 按顺序记录每个序列的模式（多模态场景下使用）

    数据加载：构建流程

    构建数据加载器是一个具备分布式感知的过程，涉及以下类：

    1. BlendedMegatronDatasetConfig
    2. BlendedMegatronDatasetBuilder
    3. IndexedDataset
    4. MegatronDataset
    5. BlendedDataset

    详细信息可参阅类文档。

    BlendedMegatronDatasetConfig（可扩展）

    BlendedMegatronDatasetConfig 用于参数化 BlendedMegatronDatasetBuilder，进而驱动 MegatronDataset 与 BlendedDataset。

    不同的训练/推理场景需要扩展该配置类，例如 GPTDatasetConfig。

    BlendedMegatronDatasetBuilder

    BlendedMegatronDatasetBuilder 用于构建 Megatron Core 中最高层的数据接口。

    注意： 所有 rank 都需要尝试通过 BlendedMegatronDatasetBuilder 构建数据集，否则程序会阻塞。具体由哪些 rank 实际执行构建，可通过 BlendedMegatronDatasetConfig 控制。

    IndexedDataset

    IndexedDataset 是 Megatron Core 最底层的数据接口，在构建高层数据接口之前，必须确保对应的 .bin/.idx 文件已存在。

    MegatronDataset（可扩展）

    MegatronDataset 抽象类是基于 IndexedDataset 的高层数据接口。不同场景可扩展出各自的数据集，例如 GPTDataset。

    BlendedDataset

    BlendedDataset 基于 MegatronDataset 提供更高层的数据接口。当需要将多个数据分布（即多个 MegatronDataset 实例）按照一定比例混合到某个数据划分时才需使用，可通过 BlendedMegatronDatasetConfig 控制混合权重。

    数据加载：实现细节

    GPTDataset

    GPTDataset 由以下变量参数化：底层 IndexedDataset 实例 indexed_dataset，划分索引 indexed_indices（训练/验证/测试所用连续文档或序列索引子集），样本数量 N，序列长度 S，随机种子 R。

    GPTDataset 创建三个索引以支持快速查找：（1）文档索引、（2）样本索引、（3）乱序索引。

    1. 文档索引 Do_idx 是长度为 E * |indexed_indices| 的一维数组（E 为最小满足 E * |indexed_indices| >= N 的 epoch 数），用于映射到文档索引，按照随机种子 R 进行乱序。

      已知：

N = 15
indexed_indices = [5, 6, 7, 8, 9]
E = 3

则示例：

Do_idx = [8, 8, 9, 6, 7, 5, 8, 5, 6, 6, 5, 9, 7, 7, 9]

    2. 样本索引 Sa_idx 是形状为 [N + 1, 2] 的二维数组，用于将样本编号 j 映射到二元组（i, Do_idx[ i ] 的偏移）。第 j 行和第 j + 1 行分别表示第 j 个样本的左右边界。

      已知：

S = 1024

则示例：

Sa_idx[0] = (0, 0)
Sa_idx[1] = (0, 1024)       => Do_idx[0] 长度大于 S
Sa_idx[2] = (1, 512)        => Do_idx[0] 长度为 1536
Sa_idx[3] = (2, 0)          => Do_idx[1] 长度为 1536
Sa_idx[4] = (5, 300)        => Do_idx[2:5] 相比 Do_idx[0:2] 更短
Sa_idx[5] = (6, 24)         => Do_idx[5] 长度为 1300

    3. 乱序索引 Sh_idx 是长度为 N 的一维数组，将 k 映射到 j，同样按照 R 进行乱序。

      已知：

N = 10

则示例：

Sh_idx = [4, 0, 2, 6, 1, 9, 5, 8, 7, 3]

    若要查询第 k 个样本：

    • 使用乱序索引获取样本索引位置 j：

      j = Sh_idx[k]
      - 使用样本索引得到对应的文档索引边界与偏移：

      i, offset = Sa_idx[j]
i_next, offset_next = Sa_idx[j + 1]
      - 根据文档索引从连续的文档中取出长度为 S 的 token：

    python sample = [] sample += indexed_dataset[Do_idx[i]][offset:] if i != i_next: sample += indexed_dataset[Do_idx[i + 1:i_next]] sample += indexed_dataset[Do_idx[i_next]][:offset_next]

    为加速初始化，各索引会在单个进程顺序构建与缓存，其余进程并行读取缓存。缓存文件名基于 MegatronDataset.__init__ 内生成的哈希值。

    BlendedDataset

    BlendedDataset 由以下变量参数化：底层 MegatronDataset 列表 D，对应权重 W（每个数据集一个），以及混合后的目标大小 S。采样时会按权重比例从各数据集抽样，直到得到指定规模的混合数据集。每次采样都会选择当前采样误差最大的那个数据集取样。

    BlendedDataset 使用两类索引：（1）数据集索引、（2）数据集样本索引。

    1. 数据集索引 Da_idx 是长度为 S 的一维数组，将 i 映射到数据集编号。

      ``` 已知

      D = [d0, d1, d2] W = [1/2, 1/4, 1/4] S = 4

      则示例：

      Da_idx = [0, 1, 2, 0] ```

    2. 数据集样本索引 Sa_idx 是长度为 S 的一维数组，将 i 映射到 Da_idx[i] 对应数据集中的样本编号。

      ``` 已知

      Da_idx = [0, 1, 2, 0]

      则示例：

      Sa_idx = [0, 0, 0, 1] ```

    若要查询第 k 个样本：

    • 使用数据集索引确定数据集，并使用数据集样本索引获取样本：

      sample = D[Da_idx[k]][Sa_idx[k]]

    同样地，这些索引会在单个进程中顺序构建与缓存，再由其他进程并行加载。缓存文件名基于 BlendedDataset.__init__ 中生成的哈希值。
