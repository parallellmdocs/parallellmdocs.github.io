    Megatron-FSDP

    [![Python 3.10+](https://img.shields.io/badge/python-3.10+-blue.svg)](https://www.python.org/downloads/release/python-3100/)

    Megatron-FSDP 简介

    Megatron-FSDP 是 NVIDIA 开发的 PyTorch 扩展，提供 Fully Sharded Data Parallelism (FSDP) 的高性能实现。它与主流深度学习框架和并行库高度兼容，可帮助你轻松在多 GPU 与多节点环境中扩展 PyTorch 模型。

    与 FSDP2 相比，Megatron-FSDP 可带来最高约 25% 的速度提升与 23% 的显存节省。

    兼容性

    • PyTorch DTensor
    • Megatron Core
    • TransformerEngine

    功能亮点

    • 易于集成：通过简单的 fully_shard 函数即可快速完成模型并行化
    • 高性能：针对 NVIDIA GPU 进行了性能优化与高效内存管理
    • 跨框架互操作：可无缝配合 PyTorch、Huggingface Transformers、Megatron-LM、Megatron Bridge 与 TransformerEngine
    • 高扩展性：同时支持单机多卡与多节点分布式训练
    • 灵活配置：可自定义分片策略与通信进程组

    性能优化

    • 高级分桶机制：基于数据类型的分桶策略，降低通信集体操作的开销
    • 缓冲区管理：通过 ParamAndGradBuffer 类重排参数与主梯度的存储，实现零拷贝通信
    • 通信与计算重叠：改进参数 all-gather 与梯度 reduce-scatter 的重叠效率
    • 基于用户缓冲注册的 NCCL 通信：将 NCCL 通信卸载到 NVL/IB SHARP，降低通信时对 GPU SM 的占用
    • 结合 Transformer Engine 的 FP8 混合精度：与 Transformer Engine 兼容，支持高效的 FP8 混合精度训练
    • 结合 Transformer Engine 的梯度累积融合：在反向传播中移除显式的梯度拷贝到通信缓冲区

    安装

    pip install megatron-fsdp
    • PyPI: https://pypi.org/project/megatron-fsdp/
    • 源码仓库: https://github.com/NVIDIA/Megatron-LM/tree/main/megatron/core/distributed/fsdp/src

    快速上手

    基本用法

    只需几行代码即可让你的 PyTorch 模型具备 Fully Sharded Data Parallelism 能力：

    import torch
from megatron_fsdp import fully_shard

# Your existing model and optimizer
model = YourModel()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

# Enable FSDP with Megatron-FSDP.
# Alternatively, you can use fully_shard_model() followed by fully_shard_optimizer()!
model, optimizer = fully_shard(
    model,
    optimizer,
    device_mesh=device_mesh, # Your global DeviceMesh.
    dp_shard_dim="dp_shard_cp", # Sharding across the flattened DP-CP mesh.
    fsdp_unit_modules=[YourTransformerBlock], # Modules to shard.
)

# Your model is now ready for distributed training!

    与 FSDP-2 的对比

    fully_shard / fully_shard_model / fully_shard_optimizer 提供了进入 MegatronFSDP 的简单入口。

    • 无需对所有子模块逐个调用 fully_shard，只要传入需要分片的子模块类或导入路径即可。
    • 通过一行配置即可切换分片方案，同时完整保留原训练循环的结构与语义。

    与 FSDP2 的用法对比：

    import torch
from torch.distributed.fsdp import fully_shard

# Your existing model and optimizer
model = YourModel()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

# Enable FSDP with FSDP2
for module in model.modules():
    if isinstance(module, YourTransformerBlock): # Sub-Modules to shard
        fully_shard(module)
fully_shard(model)

# Your model is now ready for distributed training!

    fully_shard / MegatronFSDP API - 高级功能

    import torch
from megatron_fsdp import fully_shard

# Initialize DeviceMesh.
device_mesh = torch.distributed.device_mesh.init_device_mesh(
    "cuda",
    mesh_shape=(dp_outer_size, dp_shard_size, cp_size, tp_size),
    mesh_dim_names=("dp_outer", "dp_shard", "cp", "tp"),
)
# Only relevant when using HSDP, where we also need the full DP group for data parallelism,
# This sub-mesh can be provided to distributed samplers or dataloaders.
device_mesh[("dp_outer", "dp_shard")]._flatten("dp")
# Only required if using CP. Otherwise, just pass dp_shard to FSDP.
device_mesh[("dp_shard", "cp")]._flatten("dp_shard_cp")
# Only required if using HSDP. Otherwise, don't pass hybrid_fsdp_group.
device_mesh[("dp_outer", "dp_shard", "cp")]._flatten("hsdp")
hsdp_group = device_mesh["hsdp"].get_group()

# Fully-shards your model and distributes your optimizer.
model, optimizer = fully_shard(
    # PyTorch (Root) Module
    model,
    # PyTorch Optimizer
    optimizer,
    # Device Mesh
    device_mesh=device_mesh
    # Always required for FSDP or HSDP.
    dp_shard_dim="dp_shard_cp",
    # Set this required argument to use HSDP instead of FSDP. Otherwise, set this to None.
    dp_outer_dim="dp_outer",
    # Only required for TP-sensitive models (i.e. Megatron-LM / TransformerEngine) or when using DTensor-based TP.
    # Otherwise, set this to None.
    tp_dim="tp",
    # Only required when using HSDP. Otherwise, set this to None.
    hybrid_fsdp_group=hsdp_group,
    # FSDP Sharding Strategy: no_shard (0) / optim (1) / optim_grads (2) / optim_grads_params (3)
    zero_dp_strategy=3,
    outer_dp_sharding_strategy=1,
    # Sharded Modules
    fsdp_unit_modules=[...],
    # Initialize the model on devices in shards to avoid OOM. Requires device("meta")-init for model.
    init_model_with_meta_device=True,
    # Reduce gradients in FP32.
    grad_reduce_in_fp32=False,
    # Store distributed optimization state in FP32.
    preserve_fp32_weights=True,
    # Sync parameters and gradients each step. Allows for gradient transformations after backward pass,
    # and synchronizes parameters and gradients across HSDP groups, but deactivates compute-communication
    # overlap going into the subsequent training step.
    sync_model_each_microbatch=True,
    # Preprocess state dict for DCP checkpointing. Required for Torch Distributed Checkpoint.
    preproc_state_dict_for_dcp_ckpt=True,
)

# Save model and optimizer state.
torch.distributed.checkpoint.save({"model": model.state_dict(), "optimizer": optimizer.state_dict()}, checkpoint_id=str(CKPT_DIR))

# Load model and optimizer state.
ckpt_state_dict = {"model": model.state_dict(), "optimizer": optimizer.state_dict()}
torch.distributed.checkpoint.load(state_dict=ckpt_state_dict, checkpoint_id=str(CKPT_DIR))
# model.load_state_dict(strict=False) is only necessary to ignore TE FP8 extra state
# that is missing from the DCP checkpoint but present in TEBaseModule.
# Megatron-FSDP does not support TE FP8 extra state checkpointing with DCP.
model.load_state_dict(ckpt_state_dict["model"], strict=False)
optimizer.load_state_dict(ckpt_state_dict["optimizer"])
    • zero_dp_strategy（以及 outer_dp_sharding_strategy）用于配置不同级别的零冗余数据并行策略，详情可参阅 ZeRO (Zero Redundancy Optimizer)。它通过在 DP ProcessGroup 内分布式存放模型参数、梯度与优化器状态，并在需要时按需聚合，来降低训练阶段的 CUDA 显存占用。分片越激进通信开销越大；no_shard 显存效率最低却通信效率最佳，optim_grads_params 则显存效率最高但通信开销最大。outer_dp_sharding_strategy 面向 Hybrid-Sharded Data Parallelism (HSDP) 的 “outer” DP 组（dp_outer_dim / hybrid_fsdp_group），仅支持 no_shard（DP 全量复制）与 optim（要求 zero_dp_strategy='optim_grads_params' 的优化器状态混合分片）。
    • 默认值：zero_dp_strategy 为 optim_grads_params 或 3，outer_dp_sharding_strategy 为 no_shard 或 0
    • 0 或 no_shard 表示模型未分片，显存占用与 DDP 类似
    • 1 或 optim 表示对优化器状态进行分片，类似于 ZeRO-DP 的优化器状态分片
    • 2 或 optim_grads 表示优化器状态与梯度均被分片，类似 ZeRO-2
    • 3 或 optim_grads_params 表示优化器状态、梯度与训练参数全部分片，类似 ZeRO-3
    • fsdp_unit_modules 是希望由 MegatronFSDP 完整分片的子模块类或 str 导入路径列表。
    • 当分片策略为 1、2 或 3 时为必填项。默认值为 None。
    • device_mesh 是 torch.distributed.DeviceMesh，用于结合硬件拓扑与其他并行策略向 MegatronFSDP 提供分片上下文。
    • dp_shard_dim 为 FSDP 分片所需的子网名称，通常是数据并行 (DP) 与上下文并行 (CP) 子网展平后的组合。
      • 当模型参数在反向传播期间在 DP-CP 上复制时，DP 与 CP 进程上的梯度会同步归约，并按照 DP-CP 的 world size 归一化。关于环形注意力如何在 Transformer 的注意力层与非注意力层按序列维度分片，可参考论文 Ring Attention with Blockwise Transformers for Near-Infinite Context。
    • dp_outer_dim 为 HSDP 复制或分片所需的 “outer” DP 子网名称；若指定该参数，fully_shard 会执行 HSDP。
    • tp_dim 为张量并行 (TP) 子网名称，在 Megatron-LM 或基于 DTensor 的 TP 场景中用于 (FSDP, TP) 交错分片。
      • 关于张量并行的更多信息，可参考论文 Megatron-LM: Training Multi-Billion Parameter Language Models Using Model Parallelism。
    • hybrid_fsdp_group 是一个 ProcessGroup，包含展平后的 dp_shard_dim 与 dp_outer_dim 子网上的所有 rank，用于确定权重与梯度缓冲区的 (DP-Outer, DP-Shard) 坐标系。HSDP 需要设置该参数。
    • init_model_with_meta_device 让 MegatronFSDP 在每张 CUDA 设备上以分片方式初始化 meta 设备上的模型，避免超大模型初始化时的 OOM。用户可先在 meta 设备 上构建模型（with torch.device('meta'): ...），MegatronFSDP 会按层分片并调用可定制的 module.reset_parameters，从而在运行时任何时刻都无需将完整模型驻留在显存中。
      • 默认值为 False。
      • 当 init_model_with_meta_device=True 时，用于将模型初始化到特定设备或 rank 的 device 参数会被忽略。
    • grad_reduce_in_fp32 会使用 FP32 精度归约梯度（区别于训练时常用的 BF16 或 FP8 精度）。
      • 默认值为 False。
      • 即将支持 torch.distributed.fsdp.MixedPrecisionPolicy。
    • preserve_fp32_weights 会保留一份 FP32 精度的模型参数，用于优化步骤。
      • 默认值为 True。
      • 即将支持 torch.distributed.fsdp.MixedPrecisionPolicy。
    • overlap_grad_reduce 与 overlap_param_gather 会通过异步调用与预取，将梯度 reduce-scatter 与参数 all-gather 的通信与前向、反向计算重叠。（在 no_shard 情况下不会进行参数聚合，但会重叠梯度 all-reduce。）
      • 两者默认值均为 True。
    • sync_model_each_microbatch 会在每次前向-反向完成后触发一次 wait（MegatronFSDP.finish_grad_sync()），确保梯度归约、参数释放以及优化器参数/梯度就位（用于后续的 optimizer.step()）。在 HSDP 场景下，参数与梯度会在每次训练步而非每次优化周期上于 “outer” DP 组完成 all-gather 与归约。该行为使训练循环更加透明，便于在反向后执行梯度变换并保持干净的计算/内存状态，但在需要延迟优化（例如梯度累积）的场景下会损失性能，因为无法将上一轮迭代的通信与下一轮的计算重叠。当直接使用 MegatronFSDP 类时该参数默认值为 False。
      • 对 fully_shard 默认值为 True，直接使用 MegatronFSDP 时默认为 False。
    • keep_fp8_transpose_cache_when_using_custom_fsdp 会在使用 MegatronFSDP 时保留 FP8 转置缓存。该选项会增加约「参数数量 × 1 Byte」的内存开销，但能跳过反向传播中的权重转置操作。Blackwell 架构上不会带来收益。
      • 仅在使用 Megatron-LM 时生效。
      • 默认值为 False。
    • nccl_ub 会为参数与梯度缓冲区分配并注册 NCCL user buffer。该选项可启用更高 SM 利用率的 NCCL 算法，在通信与计算重叠场景中改善性能。如果 FSDP 通信同时覆盖 NVL 与 IB 域，与 SHARP 结合可获得更多收益。启用该选项需要同时启用 fsdp_double_buffer，因此会增加额外显存占用。
      • 仅在使用 Megatron-LM 时生效。
      • 默认值为 False。
    • fsdp_double_buffer 会为 MegatronFSDP 通信所需的临时内存使用常驻的双缓冲。虽然这可能提升显存峰值占用，但在注册 NCCL user buffer (nccl_ub=True) 时为必需条件。目前仅在 GPT 等结构较为规整的模型上得到支持。
      • 仅在使用 Megatron-LM 时生效。
      • 默认值为 False。当启用 nccl_ub 时会自动改为 True。
    • preproc_state_dict_for_dcp_ckpt 会在 model.state_dict() 与 optimizer.state_dict() 上注册后处理钩子，为 torch.distributed.checkpoint.{save,load}（Torch DCP）做准备。具体来说，它为相关张量添加 __create_write_items__ 与 __create_chunk_list__ 方法，以便在保存与加载模型、优化器检查点时完成再分布。若需自定义分布式检查点方案，可关闭该选项。
      • 默认值为 True。
