    Straggler 检测

    针对 TP 组的 StragglerDetector

    megatron/core/utils.py 中的 StragglerDetector 类支持上下文管理，可用于基于 TP 组各 rank 的 RTT 找出慢节点。同时还能采集 GPU 的功耗、温度、利用率等指标，方便定位到具体 GPU（例如由硬件异常导致的拖慢）。
    该类还支持针对同一迭代的不同步骤记录时间事件，每个 rank 会持续收集数据。当在日志间隔内调用 reporter 时，会计算各指标在所有 rank 上的最小值与最大值，并按如下格式记录：

     0: INFO:megatron.core.utils:[2024-03-14 23:07:56] | MnRtt/Rnk: 3453.08ms/8 | MxRtt/Rnk: 3468.20ms/0 | MnPwr/Rnk: 601796W/8 | MxPwr/Rnk: 683801W/18 | MnTmp/Rnk: 52C/0 | MxTmp/Rnk: 65C/21 | MnUtl/Rnk: 97%/8 | MxUtl/Rnk: 100%/6 | MnClk/Rnk: 1950MHz/28 | MxClk/Rnk: 1980MHz/0 | MnDRtt/Rnk: 14.27ms/23 | MxDRtt/Rnk: 34.65ms/3 | MnEtpt/Rnk: 296.02TF/0 | MxEtpt/Rnk: 297.32TF/8

    指标说明

    每个指标前缀 Mn 或 Mx 分别表示最小值或最大值，后缀为对应的 rank。指标在日志间隔内求平均，格式为“指标名称/Rank”。常见指标如下：

    • Rtt：RoundTrip Time（本次迭代中所有被跟踪操作的耗时）
    • Pwr：GPU 功耗
    • Tmp：GPU 温度
    • Utl：GPU 利用率
    • Clk：GPU 时钟频率
    • DRtt：get_batch 延迟
    • Etpt：估算吞吐量。由实际计算吞吐量除以 Rtt 得到。由于未统计反向传播耗时，额外除以 3 作为估算值。

    命令行启用

    使用 StragglerDetector 时需传入 --log-straggler。另外还有两个可选参数，默认关闭： - --disable-straggler-on-startup：启动时是否保持禁用，默认为启用。 - --straggler-ctrlr-port：通过向 curl Rank0Host:port 发送请求即可切换启用/禁用，默认端口 65535。 - --straggler-minmax-count：若设置为大于 1 的值 N，会额外输出 N 个最低与最高 Etpt/Rank，示例： 

     0: INFO:megatron.core.utils:^^^^ Bottom 4 Ranks with lowest  Etpt(TF): 296.02/0, 296.17/2, 296.23/1, 296.23/4,
 0: INFO:megatron.core.utils:^^^^ Top    4 Ranks with highest Etpt(TF): 297.28/15, 297.28/11, 297.32/12, 297.32/8,

    StragglerDetector 编程方式

    StragglerDetector 支持上下文协议，且实现为 Singleton。 - 初始化：

    # Import and construct the detector where it will be used
from megatron.core.utils import StragglerDetector
stimer = StragglerDetector()
    • 每个 rank 需要进行一次配置：
    # Call once before the training loop starts
stimer.configure(world, rank, enabled=True, port=65545)

# Arguments:
#     world   : Total number of ranks
#     rank    : Rank for this trainer
#     mmcnt   : (Optional) Number of ranks to show for min/max Etpt
#     amp     : (Optional) Set to 3.0 if only forward timers are used
#     port    : (Optional) Control port, typically only for rank 0
#     prefill : (Optional) Number of events to pre-populate
#     enabled : (Optional) Whether collection is enabled on startup
    • 记录时间：
    # Wrap sections that need timing
with stimer:
    do_operation()

# Special case for timing get_batch
with stimer(bdata=True):
     input,... = get_batch(iterator,...)
    • 在主训练循环中记录日志：
    # Example logging inside the training loop
total_flops = 0.0
iteration = 0
# Main training loop
while training:
     iteration += 1
     do_step()
      total_flops += get_computed_flops()
      if iteration % log_interval:
         stimer.report(total_flops, log_interval)
         total_flops = 0.0
