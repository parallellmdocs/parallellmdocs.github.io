    多模态示例

    注意：该示例仍在积极开发中，后续可能发生变化。

    本示例完整展示如何预训练并指令微调基于 Llava 架构的视觉语言模型。若希望复现文末的基准成绩，请严格按步骤操作。

    示例在 A100 DGX 集群上验证：预训练与指令微调分别耗时约 1 天与 11 小时，总计使用 64 张 GPU，张量并行度设为 4（tp=4）。总体速度随 GPU 数量近似线性缩放。

    Megatron 的多模态支持仍在完善中。本示例目标不是产出最优模型（需要更多数据与模型调优），而是演示 Megatron 的多模态功能。如遇问题，欢迎提交 issue。

    环境准备

    Docker 容器

    可使用 examples/multimodal/Dockerfile 构建并运行示例所需的容器。

    语言模型

    按照 Mistral 文档 下载 Mistral-7B-Instruct-v0.3，并转换为 mcore 格式，张量并行度为 4。请使用 HuggingFace 提供的 tokenizer。

    视觉模型

    示例使用 OpenAI CLIP ViT-L/14@336px。下载权重并转换为 Megatron 可加载的格式：

    python examples/multimodal/model_converter/clip_converter.py --download-root /some/download/folder --output /some/output/folder --tensor-parallel-size 4 --use-te

    合并模型检查点

    更新路径后，运行以下脚本，将 Mistral 与 CLIP 模型合并为单个多模态检查点目录：

    examples/multimodal/combine_lm_vision_checkpoints.sh /path/to/mistral/model /path/to/clip/model /output/dir

    训练

    预训练

    1. 从 HuggingFace 下载 LLavA-Pretrain 数据集并解压图片（约 79GB）：
    git clone https://huggingface.co/datasets/liuhaotian/LLaVA-Pretrain
cd LLaVA-Pretrain
unzip images.zip
    1. 将数据转换为 webdataset 格式：
    cd <megatron-lm dir>
python examples/multimodal/convert_llava_pretrain_to_wds.py
    1. 转换为 megatron-energon 格式：
    cd <LLaVA-Pretrain dir>/wds
energon prepare ./

    根据提示输入以下内容：

    > Please enter a desired train/val/test split like "0.5, 0.2, 0.3" or "8,1,1": 9,1,0
> Do you want to create a dataset.yaml interactively? [Y/n]: Y
> Please enter a number to choose a class: 10 (VQAWebdataset)
> Do you want to set a simple field_map[Y] (or write your own sample_loader [n])? [Y/n]: Y
> Please enter a webdataset field name for 'image' (<class 'torch.Tensor'>): jpg
> Please enter a webdataset field name for 'context' (<class 'str'>): json[0][value]
> Please enter a webdataset field name for 'answers' (typing.Optional[typing.List[str]], default: None): json[1][value]
> Please enter a webdataset field name for 'answer_weights' (typing.Optional[torch.Tensor], default: None):

    1. 修改 pretrain_dataset.yaml，使其中的 path 指向 LLaVA-Pretrain/wds。

    2. 运行预训练脚本：

    cd <megatron-lm dir>
examples/multimodal/pretrain_mistral_clip.sh

    若训练顺利，可看到如下的损失曲线（批大小为 256）：

    修改批大小可能会改变曲线形态。对于 Llava 模型，损失曲线往往不能准确预测下游表现，因此需通过评测任务量化模型质量。我们计划在未来版本加入训练期间的零样本评估。

    预训练脚本可多次执行以恢复训练；恢复时会加载最新的模型、优化器与数据加载器状态。

    指令微调（SFT）

    1. 准备符合 Megatron-Energon 格式 的微调数据集（本示例不提供详细说明）。
    2. 修改 sft_dataset.yaml，使 path 指向训练与验证数据。

    执行指令微调脚本：

    examples/multimodal/sft_mistral_clip.sh

    同样可多次运行以恢复训练。

    评估

    文本生成

    运行：

    examples/multimodal/text_generation_mistral_clip.sh --input-image-path /path/to/input/images --output-path /some/output/directory \
    --model-path /path/to/model.pt --gt-path /path/to/groundtruth/file --task generation-task-name

    其中 --task generation-task-name 指定评测任务名称（如 captioning、MMMU）。

    预训练后

    COCO 图像描述

    1. 下载 COCO 2014 测试集图片：
    wget http://images.cocodataset.org/zips/test2014.zip
    1. 下载测试标注：
    wget https://storage.googleapis.com/sfr-vision-language-research/datasets/coco_karpathy_test.json

    1. 使用 --task captioning 运行生成脚本。

    2. 执行评估：

    python examples/multimodal/evaluate_coco.py --input-path /output/directory/from/generation --groundtruth-path /path/to/groundtruth/file

    对 Mistral-7B-Instruct+CLIP Llava 模型，COCO CIDEr 约为 94。

    微调后

    MMMU

    当前 MMMU 官方仓库无法通过 pip 安装，请在 examples/multimodal 下克隆：

    git clone https://github.com/MMMU-Benchmark/MMMU.git

    数据集会在 HuggingFace 上自动下载。先使用 --task MMMU 生成，再运行：

    python examples/multimodal/evaluate_mmmu.py --input-path /output/directory/from/generation

    对微调后的 Mistral-7B-Instruct+CLIP Llava 模型，MMMU 得分约为 38。
