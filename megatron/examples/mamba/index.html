    基于 Mamba 的语言模型

    介绍

    本文档是论文 An Empirical Study of Mamba-based Language Models 所使用代码的入口。

    我们通过 HuggingFace 发布了部分模型参数。main 分支已不再兼容 Mamba2-* 检查点，如需加载，可使用 论文中固定的代码快照。

    安装

    使用提供的 Dockerfile 构建并运行容器：

    docker build -t your_image_name:your_tag .
docker run --gpus all -it --rm \
  -v /path/to/megatron:/workspace/megatron \
  -v /path/to/dataset:/workspace/dataset \
  -v /path/to/checkpoints:/workspace/checkpoints \
  -w /workspace/megatron/examples/mamba \
  your_image_name:your_tag

    训练

    train.sh 演示了单节点预训练流程。通过设置 MODEL_SCALE 选择 800M 或 8B 规模。8B 混合模型架构与论文中描述的设置一致。

    文本生成

    使用 run_text_gen_server_8b.sh 可基于 8B 混合模型检查点启动文本生成服务。默认张量并行度为 1。

    若使用不同的并行设置或模型架构，请修改脚本中的参数。例如，运行 8B 纯 Mamba-2 模型时，将 --hybrid-attention-ratio 与 --hybrid-mlp-ratio 设置为 0.0（或删除该参数）。

    使用 run_text_gen_server_8b_gpt3.sh 可启动 8B 参考 Transformer 模型的生成服务。

    检查点格式

    推理时需确保模型配置与检查点一致，包括混合层配置与并行策略。

    如需将混合模型检查点转换为不同的张量/流水线并行度，可使用 hybrid_conversion.py，脚本末尾提供了示例命令。

    运行前请将 Megatron-LM 仓库根目录加入 PYTHONPATH：

    export PYTHONPATH=<path-to-megatron>:PYTHONPATH

    混合模型选项

    --hybrid-attention-ratio ATT：指定注意力层占比。例如 48 层中有 4 层注意力，则使用 --hybrid-attention-ratio 0.08。

    --hybrid-mlp-ratio MLP：指定 MLP 层占比。例如 48 层中有 24 层 MLP，则使用 --hybrid-mlp-ratio 0.5。

    • ATT + MLP 必须小于或等于 1.0。
    • 1.0 - ATT - MLP 即 Mamba 层比例。
    • ATT = MLP = 0 为纯 Mamba 模型。
    • ATT = MLP = 0.5 为 Transformer 模型。

    当 ATT 或 MLP 大于 0.0，或指定 --hybrid-override-pattern 时，日志会记录所使用的混合层模式。--hybrid-override-pattern 可用于自定义层分布。

    Mamba 与 Mamba-2

    当前代码库仅支持 Mamba-2。如需运行原始版本的 Mamba，请使用 论文中固定的代码快照。
