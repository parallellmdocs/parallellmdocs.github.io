    1. 快速开始

    示例脚本演示如何对 Megatron Core 训练得到的模型执行静态批量推理，入口位于 gpt_static_inference.py。同样的流程可扩展到 gpt_dynamic_inference.py。


    1.1 代码解析

    步骤 1：初始化模型并行及默认参数
    微批大小默认为 1。在纯张量并行场景下不会使用；在流水线并行中会在运行时根据配置计算。

    # Initialize Megatron model using the same model provider from training.
    initialize_megatron(
        args_defaults={'no_load_rng': True, 'no_load_optim': True, 'micro_batch_size': 1}
    )

    步骤 2：通过 model_provider_function 加载模型
    该函数同时兼容 MCore 与 Legacy 模型。

    # Load the model checkpoint
model = get_model(model_provider, wrap_with_ddp=False)
load_checkpoint(model, None, None)
model.eval()
model = model[0]

    步骤 3：选择推理引擎
    文本生成依赖推理引擎（包含调度器）。默认使用 Megatron Core 引擎 搭配 TextGenerationController。未来会支持 TRT-LLM 引擎。

        # Create an inference wrapper to setup the model.
    inference_wrapped_model = GPTInferenceWrapper(model, args)

    # Define a sampling loop.
    text_generation_controller = TextGenerationController(
        inference_wrapped_model=inference_wrapped_model, 
        tokenizer=tokenizer
    )

    # Create a static or dynamic inference engine.
    inference_engine = StaticInferenceEngine(
        text_generation_controller=text_generation_controller, 
        max_batch_size=args.max_batch_size
)

    步骤 4：执行文本生成
    SamplingParams 给出了推荐默认值，可按需调整 top_p、top_k、生成长度等。生成结果以 InferenceRequest 列表返回。

    results: List[InferenceRequest] = inference_engine.generate(
    prompts=args.prompts, sampling_params=sampling_params
)

if torch.distributed.get_rank() == 0:
    for idx, result in enumerate(results):
        print(f' ------------- RESULT FOR PROMPT {idx} --------------- ')
        result = {
            'id': result.request_id,
            'input_prompt': result.prompt, 
            'generated_text': result.generated_text,
            'generated_tokens' : result.generated_tokens
            }
        print(result)


    1.2 运行示例

    以下给出一个 Slurm 脚本示例。运行前请配置好 tokenizer 路径、推理参数等。关于采样参数的说明可参考 这篇博客。

    # Slurm cluster settings 
ACCOUNT=<account>
MLM_PATH=/path/to/megatron-lm
GPT_CKPT=/path/to/gpt/ckpt
VOCAB_MERGE_FILE_PATH=/path/to/vocab/and/merge/file
CONTAINER_IMAGE=nvcr.io/ea-bignlp/ga-participants/nemofw-training:23.11

srun --account $ACCOUNT \
--job-name=$ACCOUNT:inference \
--partition=batch \
--time=01:00:00 \
--container-image $CONTAINER_IMAGE \
--container-mounts $MLM_PATH:/workspace/megatron-lm/,$GPT_CKPT:/workspace/mcore_gpt_ckpt,$VOCAB_MERGE_FILE_PATH:/workspace/tokenizer \
--no-container-mount-home \
--pty /bin/bash \

# Inside the container run the following. 

cd megatron-lm/
export CUDA_DEVICE_MAX_CONNECTIONS=1

TOKENIZER_ARGS=(
    --vocab-file /workspace/tokenizer/gpt2-vocab.json
    --merge-file /workspace/tokenizer/gpt2-merges.txt
    --tokenizer-type GPT2BPETokenizer
)

MODEL_ARGS=(
    --use-checkpoint-args
    --use-mcore-models
    --load /workspace/mcore_gpt_ckpt
)

INFERENCE_SPECIFIC_ARGS=(
    --attention-dropout 0.0
    --hidden-dropout 0.0
    --num-tokens-to-generate 20
    --max-batch-size 4
)

torchrun --nproc-per-node=4 examples/inference/gpt/gpt_static_inference.py \
    ${TOKENIZER_ARGS[@]} \
    ${MODEL_ARGS[@]} \
    ${INFERENCE_SPECIFIC_ARGS[@]} \
    --prompts "prompt one " "sample prompt two" "sample prompt 3"

NOTE: Other parameters which can be customized for inference:
--temperature (Sampling temperature)
--top_k (top_k sampling)
--top_p (top_p sampling)
--num-tokens-to-generate (Number of tokens to generate for each prompt)
--inference-batch-times-seqlen-threshold (During inference, if batch-size times sequence-length is smaller than this threshold then we will not use microbatched pipelining.')
--use-dist-ckpt (If using dist checkpoint format for the model)
--use-legacy-models (If using legacy models instead of MCore models)

    上述命令列出了推理阶段常见的可调参数，包括温度、top-k、top-p、生成长度、微批阈值、是否使用分布式检查点以及是否加载旧版模型。


    2. MCore 后端执行流程

    gpt_static_inference.py 展示了静态批推理的完整流程： * mcore_engine 的 generate() 函数接收输入 prompt。 * 引擎中的 Scheduler 会将 prompt 加入 [active requests] 池（../../megatron/core/inference/inference_request.py），直至达到最大 batch，其余请求进入等待池。 * 推理引擎会持续运行至所有请求（等待+活跃）都完成。 * 活跃请求交由文本生成控制器的 generate_all_output_tokens_static_batch() 处理。 * 该函数调用 model_inference_wrappers 的 prep_model_for_inference()，并进入自回归采样循环。 * 在自回归循环中，推理包装器的 get_batch_for_context_window() 会切分输入 token 与 mask。 * 输入 token 与 mask 传入 run_one_forward_step()，调用模型 .forward() 获得 logits。 * 输出 logits 在所有流水线并行 rank间同步。 * 文本生成控制器依据采样参数计算对数概率并执行采样。 * 采样得到的 token 会追加到输入 prompt 中，供下一轮迭代使用。 * update_generation_status() 检查哪些请求已完成或满足终止条件。 * 推理循环结束后，对结果进行去标记化，并存入 InferenceRequest，同时标记为完成。 * 调度器的 update_requests_pool() 将完成的请求放入完成池，并把等待队列中的请求提升为活跃状态。


    3. 自定义推理流水线

    推理流水线支持以下层级的扩展：

    • 推理引擎：MCore 引擎支持静态与动态批，可根据需要扩展为其他后端。
    • 文本生成控制器：包含主采样循环，可调整以支持自定义分词或采样策略。
    • Inference Wrapped Model：用于适配新的模型结构。
    • 推理参数：可自定义 top_p、top_k、生成长度、温度等采样相关参数。

    3.4 修改推理参数

    文本生成使用 SamplingParams。可通过该类修改 top_p、top_k、生成长度等参数，并在推理循环中添加自定义字段：

    from megatron.core.inference.sampling_params import SamplingParams

c = SamplingParams(temperature=0.5)
c.add_attributes({'min_length':4, 'eod_id':153})
