    数据集特征（Features）

    [Features] 用于描述数据集的内部结构，并指定底层的序列化格式。更贴近使用者的角度，它记录了数据集的高层信息：列名、列类型，乃至 [ClassLabel]。可以把 [Features] 想象成数据集的骨架。

    [Features] 的结构十分简单：dict[column_name, column_type]。即列名与列类型的映射关系。通过列类型你可以表达多种数据形态。

    以 GLUE 基准中的 MRPC 数据集为例：

    >>> from datasets import load_dataset
>>> dataset = load_dataset('nyu-mll/glue', 'mrpc', split='train')
>>> dataset.features
{'idx': Value('int32'),
 'label': ClassLabel(names=['not_equivalent', 'equivalent']),
 'sentence1': Value('string'),
 'sentence2': Value('string'),
}

    其中 [Value] 特征告诉 🤗 Datasets：

    • idx 列的数据类型为 int32。
    • sentence1 与 sentence2 列的数据类型为 string。

    🤗 Datasets 还支持 bool、float32、binary 等多种数据类型。

    提示

    查看 [Value] 文档可获取完整的支持类型列表。

    [ClassLabel] 特征表示 label 列包含两个类别，名称分别为 not_equivalent 与 equivalent。在数据集中标签以整数形式存储，通过 [ClassLabel.int2str] 与 [ClassLabel.str2int] 可以在整数与标签名称之间互相转换。

    如果某个列存储对象列表，就应使用 [List] 特征。仍以 SQuAD 数据集为例：

    >>> from datasets import load_dataset
>>> dataset = load_dataset('rajpurkar/squad', split='train')
>>> dataset.features
{'id': Value('string'),
 'title': Value('string'),
 'context': Value('string'),
 'question': Value('string'),
 'answers': {'text': List(Value('string')),
  'answer_start': List(Value('int32'))}}

    answers 字段由嵌套特征构成，包含 text 与 answer_start 两个子字段，分别是 string 与 int32 的列表。

    提示

    参考 flatten 章节，了解如何将这些嵌套子字段展开成独立列。

    数组型特征适合创建不同维度的数组。例如使用 [Array2D] 创建二维数组，甚至可以用 [Array5D] 构造五维数组：

    >>> features = Features({'a': Array2D(shape=(1, 3), dtype='int32')})

    数组类型还支持首维大小可变。这在处理变长序列（如句子）时十分便利，无需额外的填充或截断：

    >>> features = Features({'a': Array3D(shape=(None, 5, 2), dtype='int32')})

    音频特征

    音频数据集的音频列类型为 [Audio]，其中包含三项关键信息：

    • array：解码后的音频波形（一维数组）。
    • path：音频文件的本地路径。
    • sampling_rate：音频的采样率。

    加载音频数据集并访问音频列时，[Audio] 会自动解码并按需重采样音频文件：

    >>> from datasets import load_dataset, Audio

>>> dataset = load_dataset("PolyAI/minds14", "en-US", split="train")
>>> dataset[0]["audio"]
<datasets.features._torchcodec.AudioDecoder object at 0x11642b6a0>

    注意

    索引音频数据集时应先指定行号再取 audio 列（dataset[0]["audio"]），避免一次性解码与重采样整个数据集。对于大型数据集，这种操作会非常耗时。

    若将 decode=False 传给 [Audio]，则不会解码音频，而是直接返回文件路径或原始字节：

    >>> dataset = load_dataset("PolyAI/minds14", "en-US", split="train").cast_column("audio", Audio(decode=False))
>>> dataset[0]
{'audio': {'bytes': None,
  'path': '/root/.cache/huggingface/datasets/downloads/extracted/f14948e0e84be638dd7943ac36518a4cf3324e8b7aa331c5ab11541518e9368c/en-US~JOINT_ACCOUNT/602ba55abb1e6d0fbce92065.wav'},
 'english_transcription': 'I would like to set up a joint account with my partner',
 'intent_class': 11,
 'lang_id': 4,
 'path': '/root/.cache/huggingface/datasets/downloads/extracted/f14948e0e84be638dd7943ac36518a4cf3324e8b7aa331c5ab11541518e9368c/en-US~JOINT_ACCOUNT/602ba55abb1e6d0fbce92065.wav',
 'transcription': 'I would like to set up a joint account with my partner'}

    图像特征

    图像数据集的图像列类型为 [Image]，用来从字节内容中加载 PIL.Image 对象。访问 image 列时，[Image] 会自动解码文件：

    >>> from datasets import load_dataset, Image

>>> dataset = load_dataset("AI-Lab-Makerere/beans", split="train")
>>> dataset[0]["image"]
<PIL.JpegImagePlugin.JpegImageFile image mode=RGB size=500x500 at 0x125506CF8>

    注意

    与音频类似，应使用 dataset[0]["image"] 的方式，先定位行再取 image 列，避免一次性解码整个数据集的图片。

    当设置 decode=False 时，[Image] 返回图像文件的路径或原始字节，而不会解码为 PIL.Image：

    >>> dataset = load_dataset("AI-Lab-Makerere/beans", split="train").cast_column("image", Image(decode=False))
>>> dataset[0]["image"]
{'bytes': None,
 'path': '/Users/username/.cache/huggingface/datasets/downloads/extracted/772e7c1fba622cff102b85dd74bcce46e8168634df4eaade7bedd3b8d91d3cd7/train/healthy/healthy_train.265.jpg'}

    视数据集情况不同，你可能会得到本地缓存文件的路径，或者直接得到字节内容。

    你也可以从 Numpy 数组构造图像数据集：

    >>> ds = Dataset.from_dict({"i": [np.zeros(shape=(16, 16, 3), dtype=np.uint8)]}, features=Features({"i": Image()}))

    此时数组会编码为 PNG（若像素精度要求更高则使用 TIFF）。

    对于 RGB、RGBA 等多通道数组，仅支持 uint8；若使用更高精度，系统会给出警告并将数组降级为 uint8。灰度图像只要 Pillow 支持即可使用任意整数或浮点精度；若精度过高，同样会触发警告，并将 int64 降至 int32、float64 降至 float32。
