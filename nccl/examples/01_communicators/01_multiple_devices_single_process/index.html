    NCCL 示例：多设备单进程

    本示例展示如何在单个进程内使用 ncclCommInitAll 为多块 GPU 创建通信器，无需依赖 MPI 或线程库。

    概述

    当满足以下条件时，可使用 ncclCommInitAll 便捷初始化通信器：

    • 所有 GPU 由同一进程管理；
    • 运行环境为单节点；
    • 不需要多进程之间的协调。

    该方式适用于追求简洁的单节点多 GPU 应用。

    示例流程

    1. 检测设备
    2. 查询当前可用的 CUDA 设备；

    3. 输出每块 GPU 的属性信息。

    4. 创建通信器

    5. 通过一次 ncclCommInitAll 调用创建全部通信器；
    6. NCCL 自动为设备分配 rank（0 到 n-1）；

    7. 不需要分发 unique ID。

    8. 信息校验

    9. 显示每个通信器的 rank 与设备映射；

    10. 确认初始化成功。

    11. 资源清理

    12. 正确销毁通信器与 CUDA 流；
    13. 演示标准的资源管理流程。

    构建与运行

    构建

    make [NCCL_HOME=<path-to-nccl>] [CUDA_HOME=<path-to-cuda>]

    使用全部可见 GPU 运行

    ./multiple_devices_single_process

    指定 GPU 运行

    # 仅使用 0、1 号 GPU
CUDA_VISIBLE_DEVICES=0,1 ./multiple_devices_single_process

    打印 NCCL 调试信息

    NCCL_DEBUG=INFO ./multiple_devices_single_process

    代码要点

    核心函数：ncclCommInitAll

    在单节点场景下，我们使用 ncclCommInitAll 一次性创建所有通信器：

    int num_gpus;      // 通过查询 CUDA 设备得到
ncclComm_t* comms;
int* devices;      // 填写要使用的 GPU ID

// 为全部设备一次性创建通信器
NCCLCHECK(ncclCommInitAll(comms, num_gpus, devices));

    这次调用会：

    • 创建 num_gpus 个通信器；
    • 自动分配 rank（0 到 num_gpus-1）；
    • 建立内部通信拓扑；
    • 无需 unique ID。

    与 ncclCommInitRank 的对比

    ncclCommInitAll 是 ncclCommInitRank 的便捷封装，效果等同于：

    ncclUniqueId id;

ncclGetUniqueId(&id);

ncclGroupStart();
for (int i = 0; i < num_gpus; i++) {
  cudaSetDevice(i);
  ncclCommInitRank(comms[i], num_gpus, id, devices[i]);
}
ncclGroupEnd();

    适用场景

    推荐使用

    • 单节点工作负载：所有 GPU 位于同一台机器；
    • 简单应用：无需多进程协调；
    • 开发/测试：快速验证 NCCL 功能。

    不建议使用

    • 多节点集群：跨节点通信需要 MPI；
    • 需要进程隔离：每块 GPU 独占独立进程时。

    性能考量

    • 优势
    • 无进程间通信，开销更低；
    • 内存管理更直接；
    • 可直接访问所有 GPU。
    • 劣势
    • 受单进程资源限制；
    • 无法扩展到多节点。

    常见问题

    1. 无法看到所有 GPU
    2. 检查 CUDA_VISIBLE_DEVICES；
    3. 确认当前用户对 GPU 有权限；

    4. 保证没有其他进程独占 GPU。

    5. 显存不足

    6. 单进程需要管理所有 GPU 的资源；
    7. 如有需要，可考虑改用多进程方案。

    后续学习

    理解本示例后，可以：

    1. 试试使用 ncclCommInitAll 的各类集体通信示例；
    2. 与基于 MPI 的多进程方案对比性能；
    3. 尝试不同的 GPU 组合运行。
