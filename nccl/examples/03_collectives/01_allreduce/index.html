    NCCL 示例：AllReduce 集体通信

    本示例演示如何在单进程管理多块 GPU 的场景下，使用 NCCL 完成最基础的 AllReduce（求和）操作。

    概述

    AllReduce 会将所有参与者的数据通过某种规约操作（如求和、最大值、最小值等）合并，并将结果广播给所有参与者。本示例让每块 GPU 提交自身的 rank 值，最终所有 GPU 都获得全局求和的结果。为简化流程，示例采用 ncclCommInitAll 完成通信器初始化。

    示例流程

    1. 检测 GPU：通过 ncclCommInitAll 为所有设备创建通信器；
    2. 初始化数据：每块 GPU 写入对应的 rank 值（GPU0→0，GPU1→1，…）；
    3. 执行 AllReduce：对所有 GPU 的数据求和，并将结果广播回各设备；
    4. 验证结果：检查每块 GPU 是否得到期望的总和 0+1+...+(n-1)。

    构建与运行

    构建

    cd examples/03_collectives/01_allreduce
make [NCCL_HOME=<path-to-nccl>] [CUDA_HOME=<path-to-cuda>]

    使用全部 GPU 运行

    ./allreduce

    指定 GPU 运行

    CUDA_VISIBLE_DEVICES=0,1,2,3 ./allreduce

    代码要点

    数据初始化

    每个 GPU 在设备侧发送缓冲区中写入自身 rank：

    float** sendbuff;
float rank_value = (float)i;
size_t size; // 要发送的 float 数量

// 分配设备内存
CUDACHECK(cudaMalloc((void **)&sendbuff[i], size * sizeof(float)));

// 先清零，再将首元素设为 rank 值
CUDACHECK(cudaMemset(sendbuff[i], 0, size * sizeof(float)));
CUDACHECK(cudaMemcpy(sendbuff[i], &rank_value, sizeof(float), cudaMemcpyHostToDevice));

    AllReduce 操作

    所有 GPU 并行执行求和操作，为避免死锁，使用 ncclGroupStart/End 包装：

    float** recvbuff;
ncclComm_t *comms;      // 在 ncclCommInitAll 中初始化
cudaStream_t *streams;  // 通过 cudaStreamCreate 创建

CUDACHECK(cudaMalloc((void **)&recvbuff[i], size * sizeof(float)));

NCCLCHECK(ncclGroupStart());
for (int i = 0; i < num_gpus; i++) {
    NCCLCHECK(ncclAllReduce(sendbuff[i], recvbuff[i], size, ncclFloat,
                            ncclSum, comms[i], streams[i]));
}
NCCLCHECK(ncclGroupEnd());

    适用场景

    • 深度学习：数据并行训练中的梯度求平均；
    • 科学计算：并行算法中的全局求和；
    • 统计分析：计算全局和、均值等；
    • 分布式算法：任何需要集体规约操作的场景。

    关键提示

    • ncclCommInitAll 让单节点多 GPU 的初始化更简单；
    • 单节点示例无需 MPI 或 pthread；
    • 建议通过 cudaMalloc 分配设备缓冲区，并使用 cudaMemset 初始化；
    • 对所有集体调用使用 ncclGroupStart/End 包裹；
    • 通信异步执行，可与计算重叠。

    常见问题

    校验失败

    处理办法：确保每块 GPU 正确写入自身 rank 值。

    显存不足

    处理办法：减小缓冲区大小或减少参与 GPU 数量。

    错误处理

    示例使用 NCCLCHECK、CUDACHECK 宏进行错误检查，遇到失败立即退出。实际项目中可根据需要设计更完善的异常处理。

    掌握 AllReduce 后，可进一步尝试： - 点对点通信：见 02_point_to_point/； - 其他集体操作：仿照本示例实现 Broadcast、Reduce、AllGather 等； - 多节点方案：结合 01_communicators 中的 MPI 示例扩展到跨节点通信。
