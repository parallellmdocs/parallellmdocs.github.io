    NCCL Tuner 插件单元测试

    本目录提供完整的单元测试，覆盖配置解析、匹配逻辑、成本表更新等核心功能。

    目录结构

    test/
├── test_plugin.c
├── Makefile
└── README.md

    构建与运行

    make test       # 构建并运行所有测试
make            # 仅构建
./test_plugin   # 手动运行

    高级用法：

    make test-memory   # 使用 valgrind 检查内存
make test-verbose  # 输出详细日志
make coverage      # 生成覆盖率报告（需 gcov）
make test-configs  # 生成示例配置文件

    覆盖范围

    1. 初始化 (test_plugin_init)：上下文创建与销毁；
    2. 配置解析 (test_config_parsing_valid/invalid)：CSV 解析、注释/空行、环境变量；
    3. 集体匹配 (test_collective_matching)：算法/协议/通道选择；
    4. 大小匹配 (test_size_matching)：区间边界、多配置；
    5. 拓扑匹配 (test_topology_matching)：单/多节点、精准/通配；
    6. 通道默认值 (test_default_channels)：-1 的处理；
    7. 注册缓冲区 (test_regbuff_matching)：注册/非注册/缺省兼容；
    8. pipeline 匹配 (test_pipeops_matching)：单/多 pipeline、缺省兼容；
    9. 兜底逻辑 (test_no_match_fallback)：无匹配时回退到 ring/simple。

    示例输出

    成功：

    PASS: test_plugin_init
...
Test Results: 9/9 tests passed

    失败：

    FAIL: test_collective_matching - Tree/Simple should have low cost
Test Results: 8/9 tests passed

    模拟 NCCL

    测试直接使用 ../nccl/ 下的头文件（tuner.h、common.h、err.h），无需完整 NCCL 安装即可基于真实接口验证逻辑。

    CI/CD 集成

    make install-test
make test && echo "Tests passed" || echo "Tests failed"

    内存与覆盖率

    make test-memory   # 检测泄漏/非法访问
make coverage      # 生成 gcov 报告

    新增测试

    在 test_plugin.c 中新增函数并在主函数注册：

    int test_new_feature() {
  TEST_ASSERT(condition, "description");
  TEST_PASS();
}

    注册：

    total++; passed += test_new_feature();

    调试

    make CFLAGS="-g -O0 -DDEBUG"
gdb ./test_plugin

    清理

    make clean

    该测试套件确保 Tuner 插件在各种配置与边界条件下都能正常工作。
