    常见问题（FAQ）

    FP8 检查点兼容性

    Transformer Engine 自 1.6 起支持 FP8 注意力，会在检查点的 ._extra_state 键下存储 FP8 元数据（缩放因子与 amax 历史）。随着 FP8 注意力从单后端扩展到多后端，该键所在的位置也发生了变化。

    以 MultiheadAttention 模块为例。在 Transformer Engine 1.11 中，其 FP8 注意力元数据位于 core_attention._extra_state，如下所示：

    >>> from transformer_engine.pytorch import MultiheadAttention, quantized_model_init
>>> with quantized_model_init(enabled=True):
...     mha = MultiheadAttention(
...         hidden_size=1024,
...         num_attention_heads=16,
...         bias=True,
...         params_dtype=torch.bfloat16,
...         input_layernorm=False,
...         fuse_qkv_params=True,
...         attention_type="self",
...         qkv_weight_interleaved=True,
...     ).to(dtype=torch.bfloat16, device="cuda")
...
>>> state_dict = mha.state_dict()
>>> print(state_dict.keys())
odict_keys(['qkv.weight', 'qkv.bias', 'qkv._extra_state', 'core_attention._extra_state', 'proj.weight', 'proj.bias', 'proj._extra_state'])

    下表汇总了各版本 Transformer Engine 的检查点保存/加载行为。

    版本 ≤ 1.5

    • 保存位置： 不写入任何 FP8 元数据（当时尚未引入 FP8 注意力）。
    • 加载行为：
    • 加载 1.5 及以下版本的检查点：不会寻找 FP8 元数据。
    • 加载更高版本（>1.5）的检查点：出现 unexpected key 报错。

    版本 1.6 ~ 1.7

    • 保存位置： core_attention.fused_attention._extra_state
    • 加载行为：
    • 目标版本 ≤ 1.5：自动将 FP8 元数据重置为默认值（缩放因子 1，amax 0）。
    • 目标版本 1.6 或 1.7：可直接加载保存的 FP8 元数据。
    • 目标版本 ≥ 1.8：出现 unexpected key，需手动处理键名或升级保存端。

    版本 1.8 ~ 1.11

    • 保存位置： core_attention._extra_state
    • 加载行为：
    • 目标版本 ≤ 1.5：依旧重置为默认值。

    • 目标版本 1.6 或 1.7：需要手动将旧键 core_attention.fused_attention._extra_state 映射为新键，否则同样回落到默认值。以 MultiheadAttention 为例：

      state_dict["core_attention._extra_state"] = (
    state_dict["core_attention.fused_attention._extra_state"]
)
del state_dict["core_attention.fused_attention._extra_state"]

    • 目标版本 ≥ 1.8：可直接加载保存的 FP8 元数据。

    版本 ≥ 1.12

    • 保存位置： core_attention._extra_state
    • 加载行为：
    • 目标版本 ≤ 1.5：重置为默认值。
    • 目标版本 ≥ 1.6：直接加载元数据（包含 NVFP4、MXFP8 等最新扩展）。
