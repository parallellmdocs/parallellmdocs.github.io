    配置文件结构

    若要启用调试功能，需要编写一个 YAML 配置文件，用于声明所需行为，例如哪些 GEMM（矩阵乘）应使用高精度而非 FP8，以及需要记录哪些统计数据。下面介绍如何组织该配置文件。

    通用格式

    一个配置文件可以包含一个或多个 section，每个 section 指定特定层与功能的设置：

    section_name_1:
  enabled: ...
  layers:
    # 指定层...
  transformer_engine:
    Feature1Name:
      enabled: ...
      # 功能参数...
    Feature2Name:
      enabled: ...
      # 功能参数...

section_name_2:
  enabled: ...
  layers:
    # 指定层...
  Feature1Name: # 如果功能未指定命名空间，则默认使用全局命名空间
    enabled: ...
    # 功能参数...

section_name_3:
  enabled: ...
  layers:
    # 指定层...
  transformer_engine:
    Feature1Name:
      enabled: ...
      # 功能参数...
    Feature2Name:
      enabled: ...
      # 功能参数...

    Section 名称可自行决定，但必须包含以下字段：

    1. enabled：是否启用该 section 下的功能。
    2. layers：该功能适用的层。每个层只能属于一个 section。
    3. 针对这些层的功能配置项。

    层的指定方式

    调试功能可通过 name 参数为层命名：

    linear = transformer_engine.debug.pytorch.Linear(in_features, out_features, name="linear1")

    配置文件会基于该名称识别层。layers 字段支持以下指定方式：

    1. layer_name_regex_pattern：使用正则表达式匹配层名称，语法与 Python re 模块一致。
    2. layer_types：提供字符串列表，只要层名包含任一字符串，即视为匹配。

    示例：

    # 示例 1：使用正则表达式筛选层
my_section:
  enabled: ...
  layers:
    layer_name_regex_pattern: 'self_attn.*'
  transformer_engine:
    (...)

# 示例 2：使用 layer type 筛选层
another_section:
  enabled: ...
  layers:
    layer_types: ['fc1', 'layernorm_linear']
  transformer_engine:
    (...)

    Transformer 层的命名

    在 Transformer Engine 中，可通过以下三种方式为层命名：

    • 初始化时传入 name=...。
    • 使用 debug_api.infer_and_assign_layer_names(model)，根据类名自动赋值。
    • 依赖默认命名（如 Layer_n，其中 n 为编号）。

    Transformer Engine 的 TransformerLayer 由多个子层组成。我们可以针对其中恰好包含一个线性层的子层应用精度调试工具。可在日志中查看这些层的名称。例如，名为 transformer_layer 的 TransformerLayer 可能包含：

    • transformer_layer.self_attn.layernorm_linear_qkv / transformer_layer.self_attn.linear_qkv / transformer_layer.self_attn.layernorm_linear_q / transformer_layer.self_attn.linear_q / transformer_layer.self_attn.linear_kv；
    • transformer_layer.self_attn.proj；
    • transformer_layer.inter_attn.*（layer_type="decoder" 时）；
    • transformer_layer.layernorm_mlp.fc1；
    • transformer_layer.layernorm_mlp.fc2。

    有些层（如 LayerNormLinear）融合了 LayerNorm 与 Linear。在精度调试中，只有其中的 Linear 部分会受到影响。

    下图展示了一个包含四个线性层的 TransformerLayer 示例，这些层都可以通过精度调试工具进行调整。

    图 1：示例 TransformerLayer 中各层的命名。最内层方块表示基础层，每个基础层包含一个线性层。不包含线性层的模块（如 DotProductAttention）未显示。

    配置文件示例

    # 禁用四个 GEMM 的 wgrad
section1:
  enabled: True
  layers:
    layer_types: [transformer_layer]
  transformer_engine:
    DisableFP8GEMM:
      enabled: True
      gemms: [wgrad]

# 禁用 layernorm_mlp 内的所有 GEMM
section2:
  enabled: True
  layers:
    layer_types: [layernorm_mlp]
  transformer_engine:
    DisableFP8Layer:
      enabled: True

# 记录 fc1 的 wgrad 统计
section3:
  enabled: True
  layers:
    layer_types: [fc1]
  transformer_engine:
    LogTensorStats:
      enabled: True
      stats: [min]
      tensors: [wgrad]
      freq: 1
      start_step: 0
      end_step: 50

    GEMM 与张量的结构化配置

    某些功能需要传入张量列表或 GEMM 列表。可以按不同方式进行描述。

    可直接传入列表：

    Feature:
  enabled: ...
  gemms: [gemm1, gemm2]
  tensors: [tensor1, tensor2]
  ...

    也可以为张量使用结构体写法：

    Feature:
  gemms: [gemm1, gemm2]
  tensors_struct:
  - tensor: tensor1
    feature_param1: value
  - tensor: tensor2
    feature_param1: value
  gemm_feature_param1: value

    同理，GEMM 也可使用结构体写法：

    Feature:
  enabled: ...
  tensors: [tensor1, tensor2]
  gemms_struct:
  - gemm: gemm1
    feature_param1: value
  - gemm: gemm2
    feature_param1: value
  gemm_feature_param1: value

    当然也可以同时对张量与 GEMM 使用结构体，此时 tensors_struct 需嵌套在 gemms_struct 中。
