    调用 Nvidia-DL-Framework-Inspect

    本节更详细地说明 Transformer Engine 与 Nvidia-DL-Framework-Inspect 的协作方式。TransformerEngine 的各个 GEMM 内部都预留了钩子（hook）。用户可以实现自定义功能类，也可以复用 TE 内置功能。config.yaml 用于描述哪些层需要启用哪些钩子。Nvidia-DL-Framework-Inspect 会结合 TE 训练流程、功能类以及 config.yaml，并负责在正确位置插入钩子。下图给出了示意：

    图 1：仅含一个 Linear 层的训练脚本示例。对于 config.yaml 中提到的张量，modify_tensor_enabled() 与 modify_tensor() 的行为会被功能类中的定义替换。其他调用则返回默认值（即不执行任何操作）。

    本页列出了 TransformerEngine 在每个 GEMM 中触发的 Nvidia-DL-Framework-Inspect API 调用。调用分为两类：

    • GEMM 调用：在每次 GEMM 时执行，用于处理或量化张量并收集信息；
    • 路由调用：在前向传播开始时执行，用于指示是否需要调用 modify_tensor() 等函数。

    若某层的所有路由调用均返回 False，则该层会使用 Transformer Engine 的融合实现，以获得最佳性能。若任意路由调用返回 True，该层将不使用融合，以便访问所有张量。需要注意的是，如果层上未启用任何功能，其性能应与未初始化 debug_api 的情况一致。

    调用 触发阶段 作用说明
    modify_tensor GEMM 在执行 GEMM 之前修改张量（如替换数值、插入假量化），由功能类具体实现。
    modify_tensor_enabled 路由 返回布尔值，指示是否需要调用 modify_tensor。若为 False，保持融合实现。
    fp8_gemm_enabled 路由 控制是否允许该 GEMM 使用 FP8 精度；当返回 False 时会回退到高精度路径。
    inspect_tensor GEMM 在 GEMM 前后收集张量信息（例如统计量、导出截断样本）。
    inspect_tensor_postquantize GEMM 针对量化后的张量执行额外检查或记录。
    inspect_tensor_enabled 路由 判断是否需要执行 inspect_tensor。
    inspect_tensor_postquantize_enabled 路由 判断是否需要执行 inspect_tensor_postquantize。
