    JAX API

    预定义的逻辑轴变量

    以下变量可从 transformer_engine.jax.sharding 模块直接获取，用于在 Mesh 中描述各个维度的分片策略。

    • BATCH_AXES：批次维度的逻辑轴，通常沿数据并行（DP）与 FSDP 方向分片。
    • SEQLEN_AXES：序列长度维度的逻辑轴，通常不进行分片。
    • SEQLEN_TP_AXES：序列长度维度的逻辑轴，通常沿张量并行（TP）方向分片。
    • HEAD_AXES：多头注意力中 head 维度的逻辑轴，通常沿 TP 分片。
    • HIDDEN_AXES：隐藏维度的逻辑轴，通常不分片。
    • HIDDEN_TP_AXES：隐藏维度的逻辑轴，通常沿 TP 分片。
    • JOINED_AXES：未显式定义的逻辑轴，通常不分片。

    检查点策略

    在 Transformer Engine JAX 中启用激活重计算（checkpointing）时，需要留意模型使用的检查点策略。若策略依赖 dot 原语，如 jax.checkpoint_policies.dots_with_no_batch_dims，则可能与 Transformer Engine 提供的 GEMM 不兼容，因为这些算子并不总是使用 jax.lax.dot_general。建议改用 transformer_engine.jax.checkpoint_policies.dots_and_te_gemms_with_no_batch_dims 等专为 Transformer Engine GEMM 与 jax.lax.dot_general 设计的策略。若策略不通过原语名称进行过滤（例如 jax.checkpoint_policies.save_only_these_names 或 jax.checkpoint_policies.everything_saveable），则可直接搭配使用。

    模块

    • transformer_engine.jax.flax.TransformerLayerType：枚举支持的 Transformer 层类型，用于在 Flax 中选择实现。
    • transformer_engine.jax.MeshResource：封装 Mesh 资源信息，帮助在 PJIT/PGJIT 中管理设备拓扑。
    • transformer_engine.jax.fp8_autocast / autocast：JAX 端启用 FP8 混合精度的上下文管理器。
    • transformer_engine.jax.update_collections：在训练循环中更新 FP8 统计量与集合。
    • transformer_engine.jax.flax.LayerNorm、DenseGeneral、LayerNormDenseGeneral、LayerNormMLP：TE 提供的 Flax 模块，均内置 FP8 缩放逻辑。
    • transformer_engine.jax.flax.RelativePositionBiases：实现相对位置偏置，用于 Transformer 注意力。
    • transformer_engine.jax.flax.DotProductAttention、MultiHeadAttention、TransformerLayer：JAX/Flax 的注意力与 Transformer 组件，适配 Mesh/Sharding 与 FP8。
    • transformer_engine.jax.flax.extend_logical_axis_rules：扩展 Flax 逻辑轴规则，确保自定义模块在 Mesh 中正确分片。
