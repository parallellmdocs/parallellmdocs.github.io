    安装

    先决条件

    .. _driver link: https://www.nvidia.com/drivers

    1. Linux x86_64
    2. CUDA 12.1+（Blackwell 需 12.8+） <https://developer.nvidia.com/cuda-downloads>__
    3. 支持 CUDA 12.1 及以上版本的 driver link_
    4. cuDNN 9.3 <https://developer.nvidia.com/cudnn>__ 或更新版本

    若运行时无法在标准路径（如 CUDA_HOME）找到 CUDA Toolkit 头文件，请在环境变量中设置 NVTE_CUDA_INCLUDE_PATH。

    NGC 容器中的 Transformer Engine

    自 22.09 起，NVIDIA GPU Cloud <https://ngc.nvidia.com>_ 发布的 PyTorch 容器已预装 Transformer Engine。

    pip（来自 PyPI）

    可直接从 我们的 PyPI <https://pypi.org/project/transformer-engine/>_ 安装 Transformer Engine，例如：

    pip3 install --no-build-isolation transformer_engine[pytorch]

    若需启用对应框架的 Python 绑定，必须在额外依赖中显式列出所需框架（逗号分隔，例如 [jax,pytorch]）。Transformer Engine 以 wheel 形式提供核心库，并为 JAX、PyTorch 扩展提供源代码打包。

    pip（来自 GitHub）

    额外先决条件

    1. 【PyTorch 支持】安装带 GPU 支持的 PyTorch <https://pytorch.org/>__。
    2. 【JAX 支持】安装带 GPU 支持的 JAX <https://github.com/google/jax/>__，版本需 ≥ 0.4.7。

    安装（稳定版）

    运行以下命令安装最新稳定版本：

    pip3 install --no-build-isolation git+https://github.com/NVIDIA/TransformerEngine.git@stable

    该命令会自动检测已安装的深度学习框架，并为其构建 Transformer Engine 支持。若需显式指定框架，可设置环境变量 NVTE_FRAMEWORK 为逗号分隔列表（例如 NVTE_FRAMEWORK=jax,pytorch）。

    安装（开发版）

    .. warning

    开发版可能包含尚未正式发布的新特性，但不在官方支持范围，不建议在生产环境使用。

    运行以下命令安装最新开发版：

    pip3 install --no-build-isolation git+https://github.com/NVIDIA/TransformerEngine.git@main

    该命令同样会自动检测已安装的框架并构建支持。若需显式指定框架，可设置 NVTE_FRAMEWORK（例如 NVTE_FRAMEWORK=jax,pytorch）。若只构建与框架无关的 C++ API，则设置 NVTE_FRAMEWORK=none。

    如需安装特定 PR（将 NNN 替换为 PR 编号）：

    pip3 install --no-build-isolation git+https://github.com/NVIDIA/TransformerEngine.git@refs/pull/NNN/merge

    安装（源码构建）

    执行以下命令从源码安装 Transformer Engine：

    # 克隆仓库、切换到 stable 分支并同步子模块 git clone --branch stable --recursive https://github.com/NVIDIA/TransformerEngine.git

    cd TransformerEngine export NVTE_FRAMEWORK=pytorch # 可选：指定要构建的框架 pip3 install --no-build-isolation . # 构建并安装

    若已克隆过仓库，请确保同步子模块：

    git submodule update --init --recursive

    可通过 “test” 选项安装额外的测试依赖：

    pip3 install --no-build-isolation .[test]

    如需使用调试符号（例如添加 -g 标志）构建 C++ 扩展：

    NVTE_BUILD_DEBUG=1 pip3 install --no-build-isolation .
