    自动混合精度示例

    .. currentmodule:: torch.amp

    在 PyTorch 中，“自动混合精度训练”通常指将 :class:torch.autocast 与 :class:torch.amp.GradScaler 搭配使用：

    • :class:torch.autocast 用于在选定代码区域开启自动混合精度，根据算子特性自动选择合适的计算精度，从而在保证数值精度的同时提升性能。
    • :class:torch.amp.GradScaler 负责梯度缩放，可显著减少 float16（CUDA/XPU 默认）梯度下溢，并在 :ref:梯度缩放<gradient-scaling> 中有详细介绍。

    两者彼此独立，可按需组合。本文档按主题列举常见用法，帮助你在工程中落地自动混合精度（示例代码仅用于说明，完整演练可参考 Automatic Mixed Precision recipe <https://pytorch.org/tutorials/recipes/recipes/amp_recipe.html>_）。

    :local:

    典型混合精度训练

    # 默认精度下创建模型与优化器
model = Net().cuda()
optimizer = optim.SGD(model.parameters(), ...)

# 训练开始前只需创建一次 GradScaler
scaler = GradScaler()

for epoch in epochs:
    for input, target in data:
        optimizer.zero_grad()

        # 在 autocast 上下文中执行前向
        with autocast(device_type="cuda", dtype=torch.float16):
            output = model(input)
            loss = loss_fn(output, target)

        # 缩放损失，反向传播生成缩放后的梯度
        scaler.scale(loss).backward()

        # scaler.step() 会先反缩放梯度，若不存在 inf/NaN，再执行 optimizer.step()
        scaler.step(optimizer)

        # 更新缩放因子供下一迭代使用
        scaler.update()

    .. _working-with-unscaled-gradients:

    与未缩放梯度协同

    scaler.scale(loss).backward() 生成的梯度均已被放大。如果需要在 backward() 与 scaler.step(optimizer) 之间读取或修改 .grad， 应先反缩放。以梯度裁剪为例，阈值应基于“原始梯度”而非放大后的值。

    调用 scaler.unscale_(optimizer) 会就地反缩放该优化器名下的参数梯度。若模型包含多个优化器（例如 optimizer2），需分别调用一次。

    梯度裁剪

    scaler = GradScaler()

for epoch in epochs:
    for input, target in data:
        optimizer.zero_grad()
        with autocast(device_type="cuda", dtype=torch.float16):
            output = model(input)
            loss = loss_fn(output, target)

        scaler.scale(loss).backward()

        # 对当前优化器关联的梯度执行反缩放
        scaler.unscale_(optimizer)

        # 反缩放后即可按常规方式裁剪
        torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm)

        scaler.step(optimizer)
        scaler.update()

    GradScaler 会记录本轮是否对指定优化器调用过 unscale_，因此 scaler.step(optimizer) 不会重复反缩放。

    .. currentmodule:: torch.amp.GradScaler

    :meth:`unscale_<unscale_>` 应在每次 :meth:`step<step>` 前最多调用一次，并且要确保该优化器负责的梯度均已累积完成。
若在两次 :meth:`step<step>` 之间调用多次，将触发 RuntimeError。

    与缩放梯度协同

    梯度累积

    设有效批量为 batch_per_iter * iters_to_accumulate（若为分布式训练，还需乘以 num_procs）。 应以“有效批量”为粒度决定是否跳过 optimizer.step()、是否存在 inf/NaN，以及何时更新缩放因子。 累积期间必须保持梯度均为同一缩放因子，否则会出现缩放梯度与未缩放梯度混合的情况，导致无法恢复原始梯度。

    scaler = GradScaler()

for epoch in epochs:
    for i, (input, target) in enumerate(data):
        with autocast(device_type="cuda", dtype=torch.float16):
            output = model(input)
            loss = loss_fn(output, target)
            loss = loss / iters_to_accumulate

        scaler.scale(loss).backward()

        if (i + 1) % iters_to_accumulate == 0:
            # 若需要裁剪未缩放梯度，可在此调用 scaler.unscale_(optimizer)

            scaler.step(optimizer)
            scaler.update()
            optimizer.zero_grad()

    梯度惩罚

    梯度惩罚通常通过 :func:torch.autograd.grad 生成梯度、求和得到惩罚项，再加到损失中。 以下展示未使用混合精度时的 L2 惩罚：

    for epoch in epochs:
    for input, target in data:
        optimizer.zero_grad()
        output = model(input)
        loss = loss_fn(output, target)

        grad_params = torch.autograd.grad(outputs=loss,
                                          inputs=model.parameters(),
                                          create_graph=True)

        grad_norm = 0
        for grad in grad_params:
            grad_norm += grad.pow(2).sum()
        grad_norm = grad_norm.sqrt()
        loss = loss + grad_norm

        loss.backward()
        optimizer.step()

    若希望结合梯度缩放，需要对传入 :func:torch.autograd.grad 的 outputs 进行缩放；生成的梯度也会被放大，因此在计算惩罚项前需手动反缩放。 另外，惩罚项属于前向计算，应处于 :class:autocast 上下文中。

    scaler = GradScaler()

for epoch in epochs:
    for input, target in data:
        optimizer.zero_grad()
        with autocast(device_type="cuda", dtype=torch.float16):
            output = model(input)
            loss = loss_fn(output, target)

        scaled_grad_params = torch.autograd.grad(outputs=scaler.scale(loss),
                                                 inputs=model.parameters(),
                                                 create_graph=True)

        inv_scale = 1.0 / scaler.get_scale()
        grad_params = [p * inv_scale for p in scaled_grad_params]

        with autocast(device_type="cuda", dtype=torch.float16):
            grad_norm = 0
            for grad in grad_params:
                grad_norm += grad.pow(2).sum()
            grad_norm = grad_norm.sqrt()
            loss = loss + grad_norm

        scaler.scale(loss).backward()

        scaler.step(optimizer)
        scaler.update()

    多模型、多损失与多优化器

    如果网络包含多个损失，需要分别调用 :meth:scaler.scale<scale>；若包含多个优化器，可以选择性地对部分优化器执行 :meth:unscale_<unscale_>， 但必须对每个优化器单独调用 :meth:scaler.step<step>。所有优化器完成更新后，仅需调用一次 :meth:scaler.update<update>。

    scaler = torch.amp.GradScaler()

for epoch in epochs:
    for input, target in data:
        optimizer0.zero_grad()
        optimizer1.zero_grad()
        with autocast(device_type="cuda", dtype=torch.float16):
            output0 = model0(input)
            output1 = model1(input)
            loss0 = loss_fn(2 * output0 + 3 * output1, target)
            loss1 = loss_fn(3 * output0 - 5 * output1, target)

        scaler.scale(loss0).backward(retain_graph=True)
        scaler.scale(loss1).backward()

        scaler.unscale_(optimizer0)

        scaler.step(optimizer0)
        scaler.step(optimizer1)
        scaler.update()

    每个优化器都会独立检查梯度中的 inf/NaN 并决定是否跳过 step()。 虽然可能出现某个优化器跳过、另一个正常更新的情况，但这种情况非常少见（数百迭代才会发生一次），通常不会影响收敛。

    .. _amp-multigpu:

    多 GPU 使用方式

    以下问题只与 :class:autocast 相关，:class:GradScaler 的用法保持不变。

    .. _amp-dataparallel:

    单进程 DataParallel

    即便 :class:torch.nn.DataParallel 会在内部为每块设备创建线程执行前向，autocast 状态仍会正确传递。如下用法是安全的：

    model = MyModel()
dp_model = nn.DataParallel(model)

with autocast(device_type="cuda", dtype=torch.float16):
    output = dp_model(input)   # 内部线程会自动套上 autocast
    loss = loss_fn(output)     # loss 同样运行在 autocast 中

    单 GPU/进程的 DistributedDataParallel

    当 :class:torch.nn.parallel.DistributedDataParallel 遵循“一进程一 GPU”的推荐配置时，不会额外创建线程，因此与单卡场景相同，直接使用 :class:autocast 与 :class:GradScaler 即可。

    单进程多 GPU 的 DistributedDataParallel

    在这种模式下，DDP 可能像 DataParallel 一样为每个设备启动侧线程。做法与前述相同：将 autocast 写进模型的 forward 内部，使侧线程自动继承该状态。

    .. _amp-custom-examples:

    Autocast 与自定义 Autograd 函数

    若网络中使用了 :ref:自定义 autograd 函数<extending-autograd>（继承 :class:torch.autograd.Function），且满足以下任意条件，则需要额外处理以兼容 autocast：

    • 函数接收多个浮点 Tensor；
    • 函数内部调用了自动可混合精度的算子（参见 :ref:Autocast 算子参考<autocast-op-reference>)；
    • 函数要求特定 dtype（例如包装了只支持单一精度的 CUDA 扩展）。

    如果函数来自外部库且无法修改，可在调用处禁用 autocast 并显式转换到 float32（或所需精度）：

    with autocast(device_type="cuda", dtype=torch.float16):
    ...
    with autocast(device_type="cuda", dtype=torch.float16, enabled=False):
        output = imported_function(input1.float(), input2.float())

    若能够修改函数定义，推荐使用 :func:torch.amp.custom_fwd 与 :func:torch.amp.custom_bwd 装饰器实现更优雅的支持。

    多输入或内部调用可混合精度算子

    在 forward 与 backward 上分别添加 :func:custom_fwd<custom_fwd> 与 :func:custom_bwd<custom_bwd>（无需参数），即可确保前向/反向与当前 autocast 状态一致：

    class MyMM(torch.autograd.Function):
    @staticmethod
    @custom_fwd
    def forward(ctx, a, b):
        ctx.save_for_backward(a, b)
        return a.mm(b)

    @staticmethod
    @custom_bwd
    def backward(ctx, grad):
        a, b = ctx.saved_tensors
        return grad.mm(b.t()), a.t().mm(grad)

    使用时无需再禁用 autocast：

    mymm = MyMM.apply

with autocast(device_type="cuda", dtype=torch.float16):
    output = mymm(input1, input2)

    需要特定精度的函数

    若函数必须在 torch.float32 下运行，可在 forward 上添加 :func:custom_fwd(device_type='cuda', cast_inputs=torch.float32)<custom_fwd> 并在 backward 上添加 :func:custom_bwd(device_type='cuda')<custom_bwd>。装饰器会在进入函数时自动转换输入，并在执行期间关闭 autocast：

    class MyFloat32Func(torch.autograd.Function):
    @staticmethod
    @custom_fwd(device_type="cuda", cast_inputs=torch.float32)
    def forward(ctx, input):
        ctx.save_for_backward(input)
        ...
        return fwd_output

    @staticmethod
    @custom_bwd(device_type="cuda")
    def backward(ctx, grad):
        ...

    func = MyFloat32Func.apply

with autocast(device_type="cuda", dtype=torch.float16):
    output = func(input)  # 函数内部将始终以 float32 执行
