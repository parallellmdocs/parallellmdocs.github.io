    DDP 通信 Hook 简介

    DistributedDataParallel（DDP）在同步梯度时默认使用 all-reduce。通信 Hook 允许用户替换这一流程，实现更高效或更灵活的通信策略。PyTorch 提供了一些内置 Hook，也支持注册自定义 Hook。

    如何注册 Hook

    在训练循环开始前，对 DDP 模型调用 register_comm_hook(state, hook)：

    ddp_model = torch.nn.parallel.DistributedDataParallel(model)
ddp_model.register_comm_hook(state=None, hook=built_in_hook)

    state 可以为 None 或任意用户自定义对象，hook 则是一个回调函数，接收 state 和 GradBucket，返回一个 torch.futures.Future。

    Hook 作用对象

    Hook 在梯度 all-reduce 前触发。为提高通信与计算重叠，DDP 会将梯度张量划分为多个 bucket，通过 torch.distributed.GradBucket 表示。Hook 可以读取或修改 bucket 中的内容，然后执行自定义通信逻辑。

    常用方法：

    • bucket.buffer()：获取扁平化后的梯度缓冲区。
    • bucket.gradients()：获取原始梯度张量列表。
    • bucket.set_buffer(tensor)：替换梯度缓冲区（需与原形状匹配）。

    内置通信 Hook

    • 默认 Hook（AllReduce）：等同于 DDP 原生行为。
    • PowerSGD Hook：先对梯度进行低秩近似，再同步矩阵，从而减少带宽占用。适合大规模通信受限的训练任务。
    • BatchNorm ReduceScatter：用于 SyncBatchNorm，将 all-reduce 替换为 reduce-scatter + all-gather，以提升吞吐。

    注册内置 Hook 的示例：

    from torch.distributed.algorithms.ddp_comm_hooks import default as comm_hooks

ddp_model.register_comm_hook(state=None, hook=comm_hooks.default_allreduce_hook)
# 或者使用 PowerSGD
state = comm_hooks.powerSGD.PowerSGDState(process_group=pg, matrix_approximation_rank=1)
ddp_model.register_comm_hook(state=state, hook=comm_hooks.powerSGD.powerSGD_hook)

    自定义 Hook 思路

    用户可编写 hook(state, bucket)，在函数内部执行任意通信或聚合逻辑，并返回包含结果张量的 Future。例如：

    def average_hook(state, bucket):
    tensor = bucket.buffer()
    fut = dist.all_reduce(tensor, group=state.process_group, async_op=True).get_future()
    fut = fut.then(lambda fut: fut.value()[0] / state.world_size)
    return fut

ddp_model.register_comm_hook(state=SimpleNamespace(process_group=pg, world_size=pg.size()), hook=average_hook)

    注意事项

    • Hook 必须返回一个 Future[Tensor]，并在完成后将结果张量写回 bucket。
    • 为保证数值正确性，应确保自定义通信最终产生与原始 all-reduce 等价的梯度（或根据算法需要进行裁剪、压缩）。
    • 若 Hook 持有有状态对象（如优化器状态），需自行处理跨 rank 的同步。

    通信 Hook 让 DDP 更易于扩展到带宽受限或特殊需求的场景。合理地组合内置 Hook 或自定义 Hook，可以显著降低通信开销并提升训练效率。
