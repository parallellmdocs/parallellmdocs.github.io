    torch.distributed.tensor

    :::{note} torch.distributed.tensor 目前处于 Alpha 阶段，仍在开发中。我们会尽量保持文档中大部分 API 的向后兼容，但如有必要仍可能调整。 :::

    PyTorch DTensor（Distributed Tensor）

    PyTorch DTensor 提供了简单灵活的张量分片原语，可透明地处理分布式逻辑，包括分片存储、算子计算以及跨设备/主机的集合通信。 借助 DTensor 可以构建不同的并行方案，并在多维分片时支持分片 state_dict 表示。

    以下是建立在 DTensor 之上的原生并行解决方案示例：

    • 张量并行
    • FSDP2
    .. automodule:: torch.distributed.tensor

    {class}DTensor 采用 SPMD（Single Program, Multiple Data）编程模型，使用户能够像编写单设备程序一样编写分布式程序，同时保持相同的收敛行为。 它通过 {class}DeviceMesh 与 {class}Placement 提供统一的张量分片布局（DTensor Layout）：

    • {class}DeviceMesh 使用 n 维数组描述设备拓扑以及集群的通信器。
    • {class}Placement 描述逻辑张量在 {class}DeviceMesh 上的分片布局。DTensor 支持三种放置方式：{class}Shard、{class}Replicate 与 {class}Partial。

    DTensor 类 API

    .. currentmodule:: torch.distributed.tensor

    {class}DTensor 是 torch.Tensor 的子类。这意味着创建 DTensor 后，可以像使用普通张量一样使用它， 包括执行各种 PyTorch 算子，且算子会按照单设备语义进行正确的分布式计算。

    除了继承自 torch.Tensor 的方法外，DTensor 还提供了一些额外接口，用于与 torch.Tensor 互相转换、重新分布（redistribute）布局、在所有设备上获取完整张量等。

    创建 DTensor 的方式

    .. currentmodule:: torch.distributed.tensor

    构造 {class}DTensor 有三种方式：

    • {meth}distribute_tensor：从逻辑（全局）torch.Tensor 创建 DTensor，可用于分片叶子张量（模型参数/缓冲区、输入等）。
    • {meth}DTensor.from_local：从每个 rank 的本地 torch.Tensor 创建 DTensor，可用于分片非叶子张量（前向/反向的中间激活）。
    • 工厂函数（如 {meth}empty、{meth}ones、{meth}randn 等）：直接指定 {class}DeviceMesh 与 {class}Placement 创建 DTensor。与 {meth}distribute_tensor 相比，这种方式会直接在设备上构建分片内存，而不是先创建逻辑张量再分片。

    从逻辑张量创建 DTensor

    torch.distributed 的 SPMD 编程模型会（通过 torchrun 等）启动多个进程执行同一程序，这意味着模型会在各进程上先初始化（例如初始化在 CPU、meta 设备或直接在 GPU 上）。

    DTensor 提供 {meth}distribute_tensor 接口，可将模型权重或张量分片为 DTensor。它会在每个进程上从“逻辑”张量创建 DTensor，确保所创建的 DTensor 拥有单设备语义，这是数值正确性的关键。

    .. autofunction::  distribute_tensor

    此外，DTensor 还提供 {meth}distribute_module，便于在 {class}nn.Module 级别上执行分片：

    随机操作

    DTensor 提供分布式 RNG 功能，保证对分片张量的随机操作生成不同值，而在复制张量上的随机操作生成相同值。 该机制要求所有参与进程在每次 DTensor 随机操作之前使用相同的随机数生成器状态；若满足此条件，操作结束后状态也会保持一致。 随机操作过程中不会执行任何同步通信。

    支持 generator 关键字参数的算子会优先使用用户传入的生成器，否则使用设备默认生成器。无论使用哪个生成器，DTensor 操作都会推进其状态。 同一个生成器既可用于 DTensor 操作，也可用于普通 Tensor 操作。但若两者混用，必须确保普通 Tensor 操作在各 rank 上推进生成器状态一致。

    当 DTensor 与管道并行同时使用时，每个管道阶段应使用不同的种子，而阶段内部的 rank 应使用相同的种子。

    DTensor 的 RNG 基于 philox 算法，可用于所有基于 philox 的后端（CUDA 及类似设备），暂不支持 CPU 后端。

    日志

    运行程序时，可以通过环境变量 TORCH_LOGS（参见 torch._logging）开启额外日志：

    • TORCH_LOGS=+dtensor：显示 logging.DEBUG 及以上级别；
    • TORCH_LOGS=dtensor：显示 logging.INFO 及以上级别；
    • TORCH_LOGS=-dtensor：显示 logging.WARNING 及以上级别。

    同时使用 Tensor 与 DTensor

    当你看到如下错误信息时：

    got mixed torch.Tensor and DTensor, need to convert all
torch.Tensor to DTensor before calling distributed operators!

    可能有两种情况。

    情况 1：用户代码错误

    最常见的错误是创建了普通张量（通过工厂函数），随后与 DTensor 混合运算，例如：

    tensor = torch.arange(10)
return tensor + dtensor

    我们不允许 Tensor 与 DTensor 混合运算：如果任一输入是 DTensor，则所有张量输入都必须是 DTensor。 这是因为其语义不明确——无法判断 tensor 在各个 rank 上是否一致。因此需要用户自行将 tensor 转换成具有正确放置方式的 DTensor。

    如果各 rank 上的 tensor 相同，可以创建一个复制的 DTensor：

    tensor = torch.arange(10)
tensor = DTensor.from_local(tensor, placements=(Replicate(),))
return tensor + dtensor

    如果需要分片 DTensor，示例如下。这意味着各分片持有张量的一部分，算子会对“完整拼接后”的数据执行：

    tensor = torch.full([], RANK)
tensor = DTensor.from_local(tensor, placements=(Shard(0),))
return tensor + dtensor

    当然，你还可以根据需求选择其他方式。

    情况 2：PyTorch 框架代码导致

    有时是 PyTorch 框架内部尝试执行 Tensor 与 DTensor 混合运算，这属于框架 bug，请提交 issue 反馈。

    对于用户而言，唯一能做的是避免触发该操作并报告问题。

    对于 PyTorch 开发者，可以通过两种途径修复：

    1. 改写框架代码，避免混合操作（类似前一部分的方式）；

    2. 在合适的位置启用 DTensor 的隐式复制。当启用后，混合操作会默认认为普通张量可复制。请谨慎使用，避免造成静默错误。
